Using Epec Edge

Epec Edge is a remote management service that can be used for edge computing, managing, monitoring and logging machine and fleet data.

 

Devices that support Epec Edge can be configured in the Edge tab.

 

 

To transfer data to Edge:

 

Data must be in the object dictionary of a device that supports Edge. Typically this means variables that the device receives from the CANopen bus. To receive data, the device has to be in the same CAN network with the other CANopen devices.

 

1. Add the device that supports Epec Edge Adding, Editing and Deleting Device

 

2. Connect the device to the network Adding, Editing and Deleting Network

 

3. Configure PDO messages Transmitting and Receiving PDOs

 

4. Select the Edge tab and configure Edge settings. The configurable properties for Edge are defined in the following table:

 

Property

Description

Timestamp OD index offset

OD index offset for timestamp of OD variable (to be sent)

Device name

Device hostname. Does not change the device’s hostname. The information is only for the edge platform and must be in sync with the current hostname.

Scope ID

Azure DPS service’s scope ID. This can be found on Azure DPS service’s “overview” page.

Docker image

Azure IoT Edge agent's image.

Username

Username to the private container registry service. If a private container registry is not being used, this field must be left blank.

Password

Password to the private container registry. If a private container registry is not being used, this field must be left blank.

Server URL

Base URL to the private container registry. If a private container registry is not being used, this field must be left blank.

Symmetric key

Use symmetric if provisioining certificate not found. Automatic: symmetric key is automatically calculated from the device serial number and the master key. Manual: the user has to calculate the symmetric key manually.

Master key

Master key for symmetric provisioning key calculation. This is available at Azure DPS service's enrollment properties.

Certificate file name

Filepath to provisioning certificate in URL format. For example, file:///opt/user/userapp/certificate.crt

Certificate key file name

Filepath to provisioning certificate key in URL format. For example, file:///opt/user/userapp/certificate.key.crt

 

Root certificate path

Filepath to provisioning certificate root certificates in URL format. For example, file:///opt/user/userapp/root_certificate.crt

 

 

5. Select the signals from which to collect the history data:

  • Click the  icon to select which OD variables are shown on the list

  • To transfer data from the system's other devices to Epec Edge, send the data in PDO message to remote device. The data must be received by the remote device to be shown on the list in Edge view.

 

6. Set the configurations for the different signals if needed. Configurations for remotely sent variables are defined in the following table:

 

Property

Description

CAN Channel

The CAN channel of the variable. Not editable.

The possible icon in front of the CAN channel number indicates

TPDO,

RPDO and

RAM variables.

Variable Name

The RAM, Retain or RetainPersistent variable name that can be sent through GlobE. Not editable.

Transfer

Select the box to transfer data to Epec Edge

Hide

Select this to exclude the variable from the System Export CSV file

Decimals

Variable value accuracy

Factor

Factor for scaling the variable

Offset

Offset for scrolling the variable

Unit

The used measurement unit (for example, ms or V).

Sampling

Variables sampling interval

Connection

Information is possible to utilize in customer application for different purposes.

For example,when cellular connection is poor, different data set is send via satellite communication.

Selection will have no effect on data transferred to Edge.

Extra Info

Extra info for the application

 

 

7. Create the project
