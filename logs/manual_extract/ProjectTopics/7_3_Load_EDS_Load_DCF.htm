Importing EDS / DCF / XDD / XDC / XML

The device's Object Dictionary and PDO configurations can be imported to EDS (Electronic Data Sheet), DCF (Device Configuration File), XDD (XML device description), or XDC (XML device configuration) files. These file types are defined by the CANopen standards (CiA DS306 and CiA DS311). 

 

The EDS and/or XDD file describes the communication behavior and object dictionary of a non-configured CANopen device.

 

The DCF and/or XDC file also includes configured settings for a specific network, such as Object Dictionary and PDO settings.

 

To import one of these files:

  1. Open the device configuration and go to the Object Dictionary tab

  2. Select the CAN channel

  3. Select icon.

  4. Browse to the file location and select Open. The configurations of Object Dictionary and PDOs are now loaded to the device and the previous OD and PDO configurations are overwritten (with the exception of  the hidden OD indexes).

 

For more information about CANopen slave, see Configuring CANopen Slave Devices / Generic Devices.
