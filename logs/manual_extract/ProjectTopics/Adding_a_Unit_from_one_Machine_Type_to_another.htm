Adding a Unit from one Machine Type to another

Adding a unit from one Machine Type to another can be done using the Include function or the Sync function.

 

  • Include: select this, to copy the unit and it’s settings to the new machine type.  

    • If changes are made to the unit, the changes will only be valid in the machine type where the unit is located.  

        

      When unit is defined as unsynchronized, the machine type code template will be generated. The selected machine type is stored in nonvolatile memory and will affect NVRAM image's layout. See Memory settings in unit's General tab for more information.

 

  • Sync: select this, to synchronize the unit to the original machine type unit.  

    • If changes are made to the unit, they will be changed in all machine types, where the unit has been synchronized to. 

       

       

      Unit types which do not support machine type code template feature are always selected as synchronized.

 

 

In network editor, select the unit to be copied. Then either, check Include or Sync beside the Machine type to be copied to. 

 

By unselecting the unit from a machine type, the unit can be removed from the machine type, or the connection between machine types can be removed. 

 

 

Synchronized units

In the device Configuration view, settings of a synchronized unit affect all the machine types where the unit is located.

 

However, in some cases the complete synchronization is not possible:

 

Section

Setting

CAN

Network Monitoring:

The control unit is not synchronized to machine types where the unit does not exist.  Note: It is possible to add units manually to network monitoring. 

 

PDO

TPDO/RPDO properties:

TPDO receiver or RPDO transmitter is not synchronized if the receiver/transmitter control unit does not exist in the machine type.

 

J1939

Message properties:

Message destination/source is not synchronized if the destination/source J1939 device does not exist in the machine type.

 

 

Unsynchronized units

Settings made for units that are not synchronized, only affect the machine type where the unit is located, and are not dependent on one another.

 

Some settings are common between machine types, even if the units are not synchronized.

 

In Network Editor, the following settings are synchronized:

Setting

Device Properties

  • Device Name

  • Device Description Version

  • CODESYS Profile

  • Maximum number of COB-IDs
