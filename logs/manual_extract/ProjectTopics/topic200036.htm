Event System and Automatically Generated Events

The purpose of Events is to send system event information, such as notifications, warnings and alarms, from producer devices to event consumer devices. The control system can have several event producers and one event consumer that receives all transmitted events. The event consumer, such as display, can log events to a file and/or show them to the user.

 

Events use PDO protocol for transmitting event information in CAN bus and therefore, two PDO messages are reserved for Events. One PDO is used to transmit event information and the other one is for acknowledge message from producer to consumer. The PDO's are generated once events are created on the Events tab.

 

To change the suggested PDO COB-ID's for event communication, open the PDO tab and change the COB-ID for the Event Send Message.

 

 

 

MultiTool Creator generates all the needed initializations for Events, such as event enumeration and global variables for connection between event producer and consumer, to the CODESYS code template PLC_PRG > Init > Init_Events action.

 

For more information about Event System:

  • Using Events

  • Epec Programming and Libraries Manual > Libraries > Common Libraries for Control Units > Event Log
