Using GlobE

Epec GlobE is a remote management service that can be used for managing, monitoring and logging machine and fleet data.

The devices that support the Epec GlobE have:

  • icon in Add Device drop-down menu and on the Device Properties

  • the GlobE tab on the device configuration

 

 

To send data to GlobE:

 

Data must be in the object dictionary of a device that supports GlobE. Typically this means variables that the device receives from the CANopen bus. To receive data, the device has to be in the same CAN network with the other CANopen devices.

 

1. Add the device that supports GlobE Adding, Editing and Deleting Device

 

2. Connect the device to the network Adding, Editing and Deleting Network

 

3. Configure PDO messages Transmitting and Receiving PDOs

 

4. Select the GlobE tab and configure GlobE settings:

  • Machine Type (used for selecting a correct machine template in Epec GlobE or creating a new one if a template with the given name doesn't exist)

  • GlobE Account Identifier (provided by Epec)

  • Send Interval (interval for sending the data to Epec GlobE server)

 

5. Select the signals from which to collect the history data:

  • Click the icon to select which OD variables are shown on the list

  • Also the device's RAM, Retain and RetainPersistent variables that are in it's OD will be shown on the list. Parameter variables can be sent to Epec GlobE in a .csv file. First, two extra columns (RemoteReadAccess and RemoteWriteAccess) needs to be added in CSV Editor. Insert value 1 to the added columns to make the parameters visible in Epec GlobE. Then include the .csv file in the application software package that is downloaded to the remote device.

  • To send data from the system's other devices to Epec GlobE, send the data in PDO message to remote device. The data must be received by the remote device to be shown on the list in GlobE view.

 

6. Set the configurations for the different signals if needed:

  • CAN Number: Remote device's CAN bus (not editable)

  • Variable Name: Variable's name (not editable)

  • Send: Select the box to collect the variable's history data

  • Hide: Select the box to exclude the variable from the System Export CSV file

  • Decimals: Accuracy

  • Unit: The used measurement unit (for example ms, text only)

  • Interval: Logging interval in minutes and seconds (mm:ss)

  • Connection: Defines the connection quality for sending the data (Low = data is sent even if the connection is poor) - not in use yet

  • GUI Text: User interface text for Epec GlobE

 

7. Create the project

 

To edit the data logging interval levels, select icon.

 

To export the signal list to a CSV file, select icon. The file is automatically created also in System Export. The CSV file defines which variables are visible in Epec GlobE and their order.

 

 

The configurable properties for GlobE are defined in the following table:

 

Property

Description

Machine Type

Defines the machine type. The machine type is used for selecting a correct machine template in Epec GlobE or creating a new one if a template with the given name doesn't exist

Account Identifier

Unique account ID for every GlobE customer (Epec provides these once the account is registered)

Send Interval

The interval for sending the data to Epec GlobE server

 

 

Configurations for remotely sent variables are defined in the following table:

 

Property

Description

CAN Channel

The CAN channel of the variable. Not editable.

The possible icon in front of the CAN channel number indicates

TPDO,

RPDO and

RAM variables.

Variable Name

The RAM, Retain or RetainPersistent variable name that can be sent through GlobE. Not editable.

Send

Select this to collect variable's history data

Hide

Select this to exclude the variable from the System Export CSV file

Decimals

Variable value accuracy

Unit

The used measurement unit (for example, ms or V).

Interval

Variable's logging interval [mm:ss].

Connection

Defines the connection quality for sending the data. Selecting Low means that the data is sent even if the connection is poor. (Not in use yet)

GUI Text

User interface text for Epec GlobE server.
