Using Diagnostics

S Series Safety Control Unit code template differs in variable names and conditions described in this chapter:

  • Code template has separate status flags for non-safety and safety application.

  • Hardware diagnostics is implemented by SafeSSeriesHardware library and it always affects safety application’s status flag.

  • Network status always affects non-safety system OK status.

  • Non-safety and safety application’s status flags both automatically affect LED-control (application override possible)

  • Output group is NOT automatically controlled on by code template

For more information on using S-Series HW diagnostics and code template, see Epec Programming and Libraries Manual.

 

To define the conditions for device diagnostics, select the Diagnostics tab. In this tab, the hardware measurement alarm limits can be set and selected as a condition for the global G_SystemOk variable. The available diagnostic information depends on the hardware type.

 

 

In addition to the selections on Diagnostics tab, the Network Status can be selected to be a condition for G_SystemOk variable on the CAN tab. (See the following picture)

 

 

MultiTool Creator generates a global variable G_SystemOk to the CODESYS code template. The selected alarm limits and error flags are conditions for the boolean G_SystemOk variable. The condition for the G_SystemOk variable can be found from PLC_PRG > EPEC_In > Diag_Action. In case the device has separate power switches for outputs, it is possible to define that they are enabled according to the G_SystemOk variable state. Also, the status LED can automatically be controlled according to the system status in devices that have LED.

Resetting Default Values

I/O Diagnostics

I/O Diagnostic Producer

 

I/O Diagnostics Producer selection enables the possibility to transfer control unit I/O pin values to CANmoon/I/O diagnostic consumer unit, which can show the values in numerical or graphical format. See more information from Epec CANmoon manual.

It is also possible select the CAN bus which is used for  I/O diagnostics.

 

 

If I/O Diagnosics Producer selection is enabled, CANopen protocol needs to be initialized on the selected CAN bus.

 

I/O Diagnostics section is only visible in CODESYS 3.5 units whose device description contains IODiagnostic and IODiagnosticTransfer libraries.

 

I/O Diagnostic Consumer

 

I/O Diagnostics Consumer selection enables the possibility to receive control unit I/O pin values which can be shown for example in I/O diagnostic window.

 

 

I/O Diagnostic consumer selection in only visible in CODESYS 3.5 units whose device description contains IODiagnostic library version 1.0.0.5 or newer. 

 

See Epec Programming and Libraries Manual, How to I/O diagnostic consumer for more information.
