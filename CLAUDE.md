## 공통 지침

1. 글로벌 지침(`~/.claude/CLAUDE.md`)과 상호 보완하여 동기화한다. 충돌 시 로컬 우선.

## MD 파일 작성 요령 (토큰 절약)

### 원칙

- **밀도 우선**: 한 줄에 최대한 많은 정보를 담는다. 도입/마무리 문장 생략.
- **서술 → 표/목록**: 문장으로 쓸 수 있는 내용은 표나 불릿으로 압축한다.
- **반복 제거**: 이미 제목·헤더에 있는 단어를 본문에서 반복하지 않는다.
- **계층 최소화**: 헤더 depth는 3단계(`###`)까지만. 더 깊으면 표로 대체.

### 구체 규칙

| 상황      | 나쁜 예                                               | 좋은 예                           |
| ----------- | ------------------------------------------------------- | ----------------------------------- |
| 항목 설명 | "이 변수는 시스템 정상 여부를 나타내는 플래그입니다." | `G_SystemOk` — 시스템 정상 플래그 |
| 조건 나열 | "A이고 B이며 C인 경우에"                              | A + B + C                         |
| 범위 표기 | "최솟값 0, 최댓값 1000"                               | `0~1000`                          |
| 단위 병기 | "밀리암페어(mA) 단위"                                 | `mA`                              |
| 코드 예시 | 전체 동작 설명 포함 블록                              | 핵심 패턴만 발췌                  |
| 주의사항  | "Note: 반드시 …해야 합니다."                          | **주의** …                        |

### 표 활용 기준

- 3개 이상 항목에 2개 이상 속성 → 표
- 항목이 2개 이하이면 불릿으로 충분
- 표 컬럼은 핵심 속성만 (최대 5열)

### 표 열 정렬 규칙

MD 표 작성 시 `|` 구분자를 세로로 정렬하여 가독성을 확보한다.

```markdown
<!-- 나쁜 예 -->
| 파라미터 | 기본값 | 단위 |
|---------|-------|------|
| MaxCurrent | 150 | mA |
| MinCurrent | 30 | mA |

<!-- 좋은 예 -->
| 파라미터       | 기본값 | 단위 |
|--------------|-------|------|
| MaxCurrent   |   150 | mA   |
| MinCurrent   |    30 | mA   |
```

적용 규칙:
- 헤더·구분선·데이터 행의 `|` 위치를 열 단위로 맞춤
- 숫자 컬럼은 오른쪽 정렬, 텍스트 컬럼은 왼쪽 정렬
- 구분선(`---`)은 해당 열 너비에 맞게 늘림

### 코드 블록 기준

- 실행 가능한 명령·선언 패턴 → 코드 블록
- 단순 파일명·변수명 → 인라인 `` ` `` 코드
- 설명용 의사 코드는 생략하고 실제 코드 사용

### 링크 활용

- 상세 내용은 별도 파일로 분리하고 링크 참조
- 본문에서 요약 1줄 + 링크로 대체

### 금지 패턴

- "위에서 설명한 바와 같이", "다음과 같습니다" 등 전환 문구
- 동일 내용을 헤더·본문·요약에 3번 반복
- 빈 섹션 (`## 개요\n\n내용 없음`)
- 과도한 강조 (`**매우** **중요한** **사항**`)

## 공통 스크립트

**규칙**: Python `.py`, 쉘 스크립트 등 외부 실행 파일은 반드시 `~/.claude/scripts/` 폴더에 생성한다.

| 스크립트           | 경로                                 | 용도              |
| -------------------- | -------------------------------------- | ------------------- |
| `md_file_skill.py` | `~/.claude/scripts/md_file_skill.py` | MD 표 열 정렬 자동 교정 |

### md_file_skill.py 사용법

```bash
# 단일 파일
python "C:\Users\JONE\.claude\scripts\md_file_skill.py" <파일.md>

# 복수 파일
python "C:\Users\JONE\.claude\scripts\md_file_skill.py" file1.md file2.md file3.md
```

- 한글/CJK 문자를 2칸으로 계산하여 열 너비 맞춤
- 코드 블록(```` ``` ````) 내부는 건드리지 않음
- 구분선(`---`)도 열 너비에 맞게 자동 조정

## Claude Code 협업 지침

### 응답 스타일

- 코드 변경 후 trailing 요약 생략 — diff로 충분
- 탐색·분석 결과는 핵심만 1~2문장
- 선택지 제시 시 권장안 1개 + 트레이드오프 1줄

### 파일 작업

- 기존 파일 우선 편집, 신규 파일 최소화
- 주석은 WHY가 명확할 때만 작성
- 문서 파일(README, *.md)은 요청 시에만 생성

### 보안·안전

- 환경변수·시크릿을 코드에 하드코딩 금지
- 파괴적 명령(rm -rf, force push 등) 실행 전 확인
- 공유 시스템 영향 작업은 사전 고지
